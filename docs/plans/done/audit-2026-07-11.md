# Coordination-system audit — 2026-07-11

Read-only multi-angle audit of the agent-coordination system (AGENTS.md, CLAUDE.md,
docs/plans/*, docs/decisions/*, scripts/plans-check.mjs, .gitignore, CI, npm lifecycle).
Method: one general pass + three independent parallel audit agents — ADVERSARY (does the
system bind?), NEWCOMER (first-task path from a bare checkout), HISTORIAN (policy vs git
reality) — 33 raw findings deduped to the list below; every critical/important claim was
re-verified against the cited file or reproduced empirically before inclusion (none proved
false; two were demoted after judgment). Findings tagged with the angles that surfaced them
independently — 2+ angles = elevated confidence.

## Verdict

**The system does not yet prevent the drift it was built to prevent.** It is a
well-shaped convention — sensible layout, a real validator whose common-case overlap
detection works (verified: `src/lib` vs `src/lib/srs.ts` IS flagged) — but nothing
executes it, nothing ties actual edits to registered claims, and its flagship product
invariant is factually inverted by the entire shipped product. Live proof on day one:
the working tree contains another agent's unregistered deck-CSV edits while
`npm run plans:check` exits green. What it demonstrably does provide: mutual visibility
between cooperating agents that voluntarily register and run the check (this audit and
drill-engine coordinated successfully). Between cooperative agents it prevents
*accidental* overlap; it prevents nothing else.

## Critical

**C1. The flagship invariant is false — and actively harmful.** [all 3 angles + general pass]
AGENTS.md:27 and ADR-001 declare "User-facing audio uses real human recordings only; never
text-to-speech." Reality: all ~6,176 committed clips are Azure neural TTS (docs/tts.md is
literally titled "Pronunciation audio (Azure TTS)"; commit caacf46 "Voice everything: full
TTS run"; e0efa27 established the pipeline 2026-06-20), src/lib/speech.ts ships a runtime
SpeechSynthesisUtterance fallback with a visible "talsyntes" marker, and the approved active
plan drill-engine.md itself blesses TTS fallback ("TTS fallback OK (words)", line 310). The
most obedient newcomer does the most wrong possible thing: refuses the documented
`npm run tts` step, treats committed assets as violations, or blocks the approved plan.
Policy that contradicts reality trains agents to override policy with judgment — the exact
failure mode a coordination system exists to remove.
→ Fix: rewrite ADR-001/AGENTS.md to the promise actually held: *pre-generated committed
da-DK neural clips are primary; runtime browser TTS only as a marked last-resort fallback;
number audio clips-only, never TTS; never run Azure synthesis without user request.*

**C2. Nothing binds: no execution point, no commit↔write_set linkage, and confinement is
not even a written rule.** [adversary + historian + general pass]
(a) plans:check runs nowhere mechanically: CI (deploy.yml) runs only astro check/test/build;
`.git/hooks` empty; pretest/prebuild run check-content.mjs. (b) The checker compares plan
files against each other only — never against the working tree or commits
(plans-check.mjs:154-181). (c) `grep -c write_set AGENTS.md CLAUDE.md` → 0 and 0: staying
inside your registered write_set is not required by any sentence anywhere. The zero-effort
drift path is: don't register, or register a tiny plan, then edit anything — no check fails,
no rule is quotably violated, and .claude/settings.local.json pre-allows
git add/commit/push, so a push deploys to production with everything green. Consequently no
commit in this repo's history has ever been subject to any coordination rule — including
the system's own adoption commit 678c6b3, which shipped 7 minutes after its plan moved to
done/ and created CLAUDE.md, a file in no write_set ever. [verified]
→ Fix: CI step running plans:check + a gate failing when changed files fall outside the
union of active write_sets; one sentence in AGENTS.md making write_set confinement a rule.

**C3. Live unregistered drift exists right now while the check passes.** [all 3 angles]
`git status`: src/data/vocab/{praksis,starter}-deck.csv modified (uncommitted, another
agent); no active plan's write_set covers src/data/vocab; plans:check exit 0. The precise
scenario the system was built for, failing on the day the system landed.
→ Fix: plans:check warns/fails on working-tree-modified files unclaimed by any active plan.

## Important

**I1. Registration is self-contradictory and cross-clone invisible.** [newcomer + historian]
AGENTS.md:21 "Approved plans are committed to active/ before implementation starts" vs
AGENTS.md:12 "Do not create commits … unless requested." This audit's own registration sat
uncommitted while the audit ran — present on one disk, reserving nothing on any other clone
(plans-check reads readdirSync, not git). Read-only tasks structurally cannot comply.
→ Fix: state that committing the registration is the one pre-authorized commit.

**I2. write_set directory semantics are disk-dependent and undocumented.** [adversary +
newcomer + general pass — three independent empirical verifications]
`normalizeWritePath` (plans-check.mjs:95) treats an entry as a directory only if it ends
with `/` or exists on disk. Existing dirs work (`src/lib` vs `src/lib/srs.ts` → flagged ✓),
but a *not-yet-created* directory without a trailing slash silently degrades to
exact-match: `src/newfeature` + `src/newfeature/engine.ts` → green. Same plan pair flips
from pass to fail when the directory is created mid-implementation. The only hint is an
unexplained `- path/` in the template — exactly the greenfield case plans are written for.
→ Fix: treat every entry as a prefix claim regardless of disk state (or hard-fail
nonexistent unslashed paths); document "directories end with /".

**I3. Staleness is dead code; the branch field is unsatisfiable.** [all 3 angles]
Age = last commit on the *declared branch* (plans-check.mjs:120-126); every plan declares
`branch: main`, shared by all agents, so anyone's commit refreshes everyone's age forever —
abandoned plans squat their write_sets indefinitely. Declaring a not-yet-created feature
branch fails validation ("branch does not exist"), while AGENTS.md:12 forbids creating
branches — `main` is the only compliant value. Bonus: 7 idle days on main fails the check
globally for everyone with no documented remedy.
→ Fix: key staleness to the plan file's own last commit (`git log -1 -- <planfile>`).

**I4. done/ is never validated and lifecycle transitions are unchecked.** [adversary + newcomer]
The validator reads only active/ (the string "done" appears nowhere in it). Setting
`status: done` while still in active/ fails the gate for everyone; nothing verifies a plan
leaving active/ arrives in done/; no abandonment procedure exists; releasing your claims
mid-conflict by moving to done/ (and continuing to edit) violates nothing quotable.
→ Fix: validate done/ frontmatter; document the atomic move (git mv + status edit, one
commit); add an abandoned-plan procedure.

**I5. Path-comparison holes: case, globs, `.`.** [adversary, empirical + code-verified]
Exact-string compare on a case-insensitive volume (`src/lib/Speech.ts` vs
`src/lib/speech.ts` → green while addressing the same file); `src/lib/*.ts` validates but
intersects nothing (globs aren't rejected or expanded); `.` validates as a whole-repo claim
that intersects nothing (no path starts with `./`).
→ Fix: casefold before comparison; reject glob metacharacters and `.`.

**I6. Plans in subdirectories of active/ are invisible.** [adversary, empirical]
`readdirSync` doesn't recurse; `active/sub/hidden.md` is neither listed nor overlap-checked
yet is quotably compliant with "Store all project planning Markdown under docs/plans/".
→ Fix: recurse, or fail on any non-plan entry in active/.

**I7. The system cannot protect itself.** [adversary]
scripts/plans-check.mjs, AGENTS.md, and every plan file are unprotected ordinary files;
owner is a free string bound to nothing (all commits share one git author). No active plan
claims scripts/ or AGENTS.md, so gutting the validator alongside drift is mechanically
unnoticed.
→ Fix: CI-run the check from a protected context; flag commits touching
plans-check.mjs/AGENTS.md/foreign plan files alongside source changes.

**I8. The frontmatter dialect exists only in the validator's source.** [newcomer, empirical]
`write_set: [a, b]` (normal YAML) fails with the misleading "write_set must contain at
least one path"; no doc specifies accepted syntax, field meanings (owner? verify? branch?),
or naming (CLAUDE.md says `<title>.md`; every real file is a kebab slug).
→ Fix: a commented example plan or docs/plans/README.md specifying the dialect and fields.

**I9. Check failures are global with no recovery protocol.** [newcomer]
Any two other agents' overlap (or staleness) makes the mandated start-of-task command exit 1
for an uninvolved newcomer; owner labels (main2-non-workflow, codex-root, claude-main) have
no resolution rule.
→ Fix: one AGENTS.md paragraph: fix your own plan; proceed + flag if the failure is between
others and your write_set is clean.

**I10. Discovery holes.** [newcomer + adversary]
No root README; docs/README.md never mentions the workflow or plans:check; .claude/ is
gitignored yet carries a broad permission allowlist (git push/reset pre-approved) invisible
to and different on every clone; humans and non-Claude/non-Codex agents get no signal.
→ Fix: root README with the workflow as its first section.

**I11. ADR-002 was false at authoring.** [historian + general pass]
"Exactly one animated moment site-wide" — three @keyframes exist (CelebrationFlag
cloth-wave ✓ sanctioned; Waveform wf-pulse on every audio play; FlashcardReviewer fade),
one introduced by the very commit (5b00b11) the corresponding done-plan celebrates, plus
sanctioned transitions. As written, routine playback violates an ADR.
→ Fix: amend to "one celebration animation"; whitelist the functional micro-animations.

**I12. The historical record under-reports itself.** [historian + general pass, verified]
Backfilled done-plan write_sets miss files their own commits touched: danish-visual-identity
lists 9 of 5b00b11's 19 files (missing 10, verified by comm against git show);
hero-real-waveforms misses package-lock.json; speaking-roadmap omits ~10 source files plus
the deck CSVs and thousands of public/audio files its cited commits changed.
→ Fix: regenerate done write_sets from `git show --name-only`; require reconciliation at
move-to-done time.

## Invariants enforced by nothing

1. "Real human recordings only; never TTS" (AGENTS.md:27, ADR-001) — unenforced AND false.
2. "Exactly one accent color / one animated moment" (ADR-002) — unenforced; false as written.
3. "Single SRS store" (ADR-003) — true today; nothing prevents a parallel store.
4. "Every task begins by running plans:check" (AGENTS.md:20) — no mechanism.
5. "Approved plans committed to active/ before implementation" (AGENTS.md:21) — no
   mechanism; contradicted by AGENTS.md:12 and by practice (678c6b3, this audit).
6. "Finished plans move to done/ with status: done" (AGENTS.md:22) — done/ never validated.
7. Write-set confinement — not even written down, let alone enforced.
8. "Plan Markdown only under docs/plans/" (AGENTS.md:16) — no check.
9. "No commits/branches unless requested" (AGENTS.md:12) — unenforceable by tooling.
10. "Temp files under /tmp, never the repo" (AGENTS.md:15) — no check (stray empty dir
    exists in-repo today).

## Top 3 fixes

1. **Create one enforcement point.** CI job (plus optional pre-commit hook):
   `npm run plans:check` AND fail if the commit's changed files ⊄ union of active
   write_sets (with a carve-out for docs/plans/** lifecycle commits). This single change
   converts the whole system from courtesy to contract.
2. **Make policy true.** Rewrite ADR-001/AGENTS.md's audio invariant to the real promise
   (committed neural clips primary; marked runtime fallback; numbers clips-only) and soften
   ADR-002 to "one celebration animation." False rules teach agents to ignore rules.
3. **Harden the validator's semantics.** Prefix-claims regardless of disk existence,
   casefold, reject globs/`.`/duplicate keys, recurse or reject subdirs in active/,
   validate done/, per-plan staleness — and write the write_set-confinement sentence plus a
   failure-recovery paragraph into AGENTS.md.
